Webで表示されるフォントは使用しているPCに依存する
MacOSであれば、「font book」というアプリに含まれている

フォントを指定しないと、表示するユーザーの環境に依存になるため、
可能な限り統一する必要がある
指定方法
```CSS
h1 {
	/* 明朝体 */
	font-family: serif;
	
	/* ゴシック体 */
	font-family: sans-serif;
	
	/* 複数指定 */
	font-family: "ヒラギノ角ゴシック", "Hiragino Kaku Gothic","遊ゴシック", "Yu Gothic", "メイリオ","Meiryo", "san-serif";
}
```
複数指定の場合は、左から順番に読み込んで、Webサイトを開こうとしているユーザーの環境が持つフォントが見つかったらそのフォントを使用するようにできている。

Q：なぜ英語と日本語を併記するのか？

A：英語圏の場合は、ローマ字で登録されている可能性があるため、併記することは必須

Q：使用したいフォントがあればユーザーにフォントを追加して貰えばいいのでは？

A：二つの理由からいい案ではない
1. 日本語は英語に比べて文字数が多く、1つのフォントを追加するのは容量が大きく現実的ではない
2. フォントの再配布となるため、ライセンスの問題でユーザーに配布することは難しい

### Webフォント
代表例：Google Fonts
https://fonts.google.com/
汎用的に使用されているフォントでは拡張性に欠ける
→Webフォントを使用することでこれを改善しようという流れがある
Web、Android、iOS、Flutterのやり方がある

**注意事項**
日本語の場合はフォントのデータ量が多い。
→Webフォントからフォントデータを読み込む時間分だけ、標準フォントの表示になる

**おすすめの使い方**
基本は標準フォントを使用する
どうしても標準フォントで対応できない
